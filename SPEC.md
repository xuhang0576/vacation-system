# 班组成员休假管理系统 - 完整设计方案

## 1. 项目概述

### 1.1 项目背景
- **项目名称**：班组成员休假管理系统
- **项目类型**：Web 应用（前后端分离）
- **核心功能**：记录、管理班组成员休假申请，支持审批流程
- **目标用户**：20人规模的班组团队

### 1.2 核心需求
| 需求 | 描述 |
|-----|------|
| 用户管理 | 注册、登录、角色区分（成员/管理员） |
| 休假申请 | 成员提交休假申请（类型、日期、理由） |
| 审批流程 | 管理员审批成员的休假申请 |
| 数据隔离 | 成员只能看自己的数据，管理员可看全部 |
| 假期统计 | 显示剩余假期额度 |

## 2. 技术架构

### 2.1 技术选型

| 层级 | 技术 | 说明 |
|-----|------|------|
| 前端 | Vue 3 + Element Plus | UI 框架，组件丰富，上手容易 |
| 后端 | Node.js + Express | 轻量级 Node 框架 |
| 数据库 | SQLite | 单文件数据库，无需安装 |
| 认证 | JWT | 无状态 Token 认证 |
| 部署 | Vercel + Railway | 免费额度可用 |

## 3. 数据库设计

### 3.1 数据表结构

#### 用户表 (users)
| 字段 | 类型 | 说明 |
|-----|------|------|
| id | INTEGER | 主键，自增 |
| username | TEXT | 用户名（唯一） |
| password | TEXT | 密码（加密存储） |
| name | TEXT | 真实姓名 |
| role | TEXT | 角色：member / admin |
| vacation_days | INTEGER | 年假天数（默认 10） |
| created_at | DATETIME | 创建时间 |

#### 休假记录表 (vacation_records)
| 字段 | 类型 | 说明 |
|-----|------|------|
| id | INTEGER | 主键，自增 |
| user_id | INTEGER | 申请人ID（外键） |
| type | TEXT | 假别：年假/病假/事假/其他 |
| start_date | DATE | 开始日期 |
| end_date | DATE | 结束日期 |
| days | REAL | 时长（天） |
| reason | TEXT | 申请理由 |
| status | TEXT | 状态：pending/approved/rejected |
| approver_id | INTEGER | 审批人ID（外键） |
| approve_time | DATETIME | 审批时间 |
| approve_note | TEXT | 审批备注 |
| created_at | DATETIME | 申请时间 |

## 4. API 设计

### 4.1 认证接口

| 方法 | 路径 | 说明 | 权限 |
|-----|------|------|------|
| POST | /api/auth/register | 注册 | 公开 |
| POST | /api/auth/login | 登录 | 公开 |
| GET | /api/auth/me | 获取当前用户信息 | 需登录 |

### 4.2 用户接口

| 方法 | 路径 | 说明 | 权限 |
|-----|------|------|------|
| GET | /api/users | 获取用户列表 | 管理员 |
| GET | /api/users/:id | 获取指定用户 | 管理员/本人 |
| PUT | /api/users/:id | 更新用户信息 | 管理员/本人 |
| PUT | /api/users/:id/role | 修改用户角色 | 管理员 |

### 4.3 休假接口

| 方法 | 路径 | 说明 | 权限 |
|-----|------|------|------|
| GET | /api/vacations | 我的休假记录 | 本人 |
| GET | /api/vacations/all | 所有休假记录 | 管理员 |
| GET | /api/vacations/pending | 待审批列表 | 管理员 |
| POST | /api/vacations | 提交休假申请 | 成员 |
| PUT | /api/vacations/:id | 更新申请 | 本人/管理员 |
| PUT | /api/vacations/:id/approve | 审批通过 | 管理员 |
| PUT | /api/vacations/:id/reject | 审批拒绝 | 管理员 |
| DELETE | /api/vacations/:id | 删除申请 | 本人/管理员 |

## 5. 页面设计

### 5.1 页面结构

```
├── 登录页 (/login)
├── 注册页 (/register)
├── 成员端 (布局: 侧边栏 + 主内容区)
│   ├── 我的休假 (/member/vacations)
│   │   └── 申请休假（对话框）
│   ├── 申请历史 (/member/history)
│   └── 假期余额 (/member/balance)
└── 管理员端 (布局: 侧边栏 + 主内容区)
    ├── 待审批 (/admin/pending)
    ├── 全部记录 (/admin/records)
    ├── 成员管理 (/admin/members)
    └── 数据导出 (/admin/export)
```

### 5.2 登录页
- Logo + 系统名称
- 用户名输入框
- 密码输入框
- 登录按钮
- "还没有账号？注册" 链接

### 5.3 注册页
- 用户名输入框（唯一）
- 密码输入框
- 确认密码输入框
- 真实姓名输入框
- 注册按钮
- "已有账号？登录" 链接

### 5.4 成员端 - 休假申请
- 假期类型下拉框（年假/病假/事假/其他）
- 开始日期选择器
- 结束日期选择器（自动计算天数）
- 申请理由文本框
- 提交按钮
- 右侧展示：当前假期余额

### 5.5 成员端 - 申请历史
- 表格展示：
  - 假别 | 开始日期 | 结束日期 | 时长 | 理由 | 状态 | 审批时间
- 状态标签着色：
  - 待审批：橙色
  - 已通过：绿色
  - 已拒绝：红色

### 5.6 管理员端 - 待审批
- 卡片式列表，每个卡片显示：
  - 申请人姓名
  - 假别和时长
  - 申请时间
  - 申请理由
  - 通过 / 拒绝按钮

### 5.7 管理员端 - 全部记录
- 高级筛选：按姓名、状态、日期范围
- 表格展示所有申请
- 可导出为 Excel

## 6. 权限设计

### 6.1 角色说明

| 角色 | 权限 |
|-----|------|
| member（成员） | 查看/提交自己的申请、查看自己的余额 |
| admin（管理员） | 成员所有权限 + 审批他人申请、查看所有记录、管理成员 |

### 6.2 数据隔离规则

```
成员视角：
  - GET /api/vacations → 只返回 user_id = 当前用户
  - GET /api/users/:id → 只能查看自己

管理员视角：
  - GET /api/vacations/all → 返回所有记录
  - GET /api/vacations/pending → 返回待审批列表
  - PUT /api/vacations/:id/approve → 可审批任意记录
```

## 7. 安全性设计

### 7.1 认证
- 密码使用 bcrypt 加密存储
- JWT Token 有效期 7 天
- Token 放在 HTTP Header: `Authorization: Bearer <token>`

### 7.2 校验
- 所有输入进行合法性校验
- 防止 SQL 注入（使用参数化查询）
- 防止 XSS（前端转义）

### 7.3 接口鉴权
- 登录接口公开
- 其他接口需要有效 Token
- 管理员操作验证角色

## 8. 部署方案

### 8.1 部署方案：全部在云服务器

**前端 + 后端 + 数据库 全部部署在同一台服务器**

| 组件 | 部署方式 |
|-----|---------|
| 前端 | Nginx 托管静态页面 |
| 后端 | PM2 守护 Node.js 进程 |
| 数据库 | SQLite 文件（项目目录下）|
| 域名/IP | 服务器公网 IP 或域名 |

### 8.2 端口规划

| 端口 | 服务 |
|-----|------|
| 80/443 | Nginx（前端静态页面） |
| 3000 | Node.js 后端 API |
